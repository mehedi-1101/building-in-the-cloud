--- PAGE 4 ---
Request Flow
REQUEST FLOW
1 User hits the public API Gateway URL in a browser or curl
2 API Gateway pass the full request to the ALB DNS on port 80
3 ALB round-robins the request to one of the 2 healthy EC2 instances in the target group
4 EC2 instance (running a web server) serves the HTML page containing its own instance ID and AZ
5 Response travels back through ALB → API Gateway → User · HTTP 200
Tip — proving round-robin works
Refresh the URL several times. The instance ID in the response should alternate between your two EC2 instances — that confirms the ALB, ASG,
target group, and both servers are all wired up correctly.
